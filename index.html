Git is easy :)

Git — розподілена система керування версіями файлів та спільної роботи.Git сприймає свої дані більш як набір знімків мініатюрної файлової системи.

Кожен раз, коли ви робите комміт, тобто зберігаєте стан вашого проекту в Git, він запам’ятовує як виглядають всі ваші файли в той момент і зберігає посилання на цей знімок.

Більшість операцій у Git потребують лише локальних файлів та ресурсів для здійснення операцій – немає необхідності в інформації з інших комп’ютерів вашої мережі.

Будь-що в Git, перед збереженням, отримує контрольну суму, за якою потім і можна на нього посилатися. Таким чином, неможливо змінити файл чи директорію так, щоб Git про це не дізнався. Цей функціонал вбудовано в систему на найнижчих рівнях і є невід’ємною частиною її філософії. Ви не можете втратити інформацію при передачі чи отримати пошкоджений файл без відома Git.Фактично, Git зберігає все не за назвою файлу, а саме за значенням хешу його змісту.

Start GIT

git init - ініціалізує(підключає) систему контролю версій в обраній папці;
git config --global user.name "my name" - вказує глобальні налаштування імені власника (для всіх репозиторіїв на пк по замовчуванню);
git config --global user.email "email@email.com" - вказує глобальні налаштування поштової адреси власника (для всіх репозиторіїв на пк по замовчуванню);
git config user.name "my name" - вказує локальні для конкретного репозиторію налаштування імені власника (необхідно виконати в папці репозиторію);
git config user.email "email@email.com" - вказує локальні для конкретного репозиторію налаштування поштової адреси (необхідно виконати в папці репозиторію);
git status - виводить в консоль інформацію по статусу, в якому перебувають файли у папці;

Add first files

git add filename.html - додає до індексування (відстеження змін) файл. filename.html-імя вашого файлу;
git add . - додає до індексування(відстеження змін) всі файли що;
git commit -m "your text info" - виконує commit файлів на пк і додає текст підказку/пояснення (text info) про те що коммітиться;

Work with GIT logs

git log - показує історію коммітів.Щоб вийти із історії необхідно натиснути CTRL+z;
git log -- pretty=online - виводить в одну строку список комітів;
git log -- pretty=online --max-count=3 - показує log (історію) останніх троьх коммітів.Щоб вийти із історії необхідно натиснути CTRL+z;
git log -- pretty=online --all - показує log (історію) всіх коммітів.Щоб вийти із історії необхідно натиснути CTRL+z;
git log -- pretty=online --author="name" - показує log (історію) всіх коммітів по конкретному імені автора.Щоб вийти із історії необхідно натиснути CTRL+z;
git log -- pretty=format:"%h - %s : %ad [ %an ]" --date=short - виведе інфу із коротким hesh по комітах,коментами,датою та автором;

Recovery and return in GIT

git checkout heshNumber - перекине нас на коміт якому відповідає номер хеша (heshnumber). Номер хеша можна взнати чезе git log,така команда використовується якщо треба вернутись на якийсь попередній коміт при дебагінгу коду тощо;
git checkout index.html - даною командою ми вертаємо файл index.html до стану в якому він був у попередньому комміті;
git reset - команда яка вертає стан файлів,гілок тощо до того стану який ми хочемо ;
git reset HEAD text.txt - команда яка вертає стан index.txt до попереднього стану,який у нього був перед git add;
git reset heshNumber text.txt - команда яка вертає стан файлу index.txt до того стану який він мав у попередньому коміті;
git revert HEAD --no-edit - вертає (попередньо видаливши наш останній коміт) нас на попередній комміт.Замість HEAD можна писати хеш якогось комміту.Видалений комміт залишається в логах;
